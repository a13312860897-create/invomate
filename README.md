# 发票管理 SaaS 项目

## 项目目标
为欧美自由职业者、小型远程团队、创业公司提供简洁易用的发票生成与管理工具，重点解决「使用复杂」「跨语言/币种受限」「缺乏自动化」等痛点。

## 技术栈
- **前端**: React + Tailwind
- **后端**: Node.js (Express / NestJS)
- **数据库**: PostgreSQL
- **认证**: JWT 或 OAuth（后期支持 Google 登录）
- **PDF 生成**: pdfkit / jsPDF
- **发邮件**: SendGrid / Postmark
- **部署**: Vercel（前端） + Railway/Render/Heroku（后端）

## 核心功能规划（MVP 优先级）

### 1. 基础功能（第一阶段必须完成）
- **用户管理**
  - 注册 / 登录 / 基本资料
  - 简单的客户（Client）管理模块
- **发票功能**
  - 创建、编辑、删除发票
  - 自动生成发票编号
  - 支持导出 PDF（含基础样式）
- **自定义选项**
  - 上传 Logo
  - 选择颜色、字体（基础 2–3 套主题即可）
- **支付状态追踪**
  - 已支付 / 未支付

### 2. 增强功能（第二阶段）
- **发票发送**
  - 通过 Email 发送，并跟踪投递状态（例如使用 SendGrid API）
- **多语言支持**
  - 英语（必备）
  - 1–2 个额外语言（如西班牙语 / 德语）
- **多币种**
  - 至少支持 USD / EUR / GBP

### 3. 未来扩展功能（第三阶段，非MVP）
- AI 自动生成发票（从邮件/文档提取数据）
- 移动端 App（Flutter/React Native）
- WhatsApp / SMS 发票推送
- 报表与统计
- Webhook / Zapier / Make 集成

## 开发里程碑（建议 3–4 周完成 MVP）

### 第 1 周：
- 项目框架搭建（前后端）
- 用户注册/登录
- 客户管理基础模块

### 第 2 周：
- 发票 CRUD
- 自动编号
- PDF 导出

### 第 3 周：
- 发票自定义（Logo/颜色/字体）
- 支付状态追踪
- 基础多语言（i18n 配置，至少英文/西语）

### 第 4 周：
- Email 发送集成（SendGrid）
- 测试与优化
- 部署上线（邀请小范围用户试用）
